## What is CedarVim?

CedarVim is a Neovim configuration that builds upon the educational foundation of [kickstart.nvim](https://github.com/nvim-lua/kickstart.nvim) and brings it to the level of a full-featured daily driver. Where Kickstart teaches you what every line does, CedarVim has already done the heavy lifting вҖ” consolidating plugins, tuning performance, and configuring a complete IDE experience out of the box.

It follows a **Unification Strategy**: instead of installing a plugin for every problem, it consolidates related functionality into single, high-performance engines. Less to maintain. Less memory. More speed.

---

## вңЁ Features

- **вҡЎ Instant Navigation** вҖ” A unified, Rust-powered search and picker engine (`snacks.nvim`) handles file finding, live grep, buffer switching, and LSP navigation with sub-millisecond response.
- **рҹҡҖ Zero-Lag Autocompletion** вҖ” `blink.cmp` provides asynchronous, Rust-based completions with Copilot ghost-text, LuaSnip snippets, and Copilot inline suggestions.
- **рҹ§  Full LSP Suite** вҖ” Automatic server installation and management via Mason. Pre-configured for HTML, CSS, TailwindCSS, TypeScript, and Lua out of the box.
- **рҹӨ– AI Integration** вҖ” `CodeCompanion.nvim` provides a persistent, dockable AI chat window and inline refactoring actions. Chat sessions can be saved, loaded, and deleted.
- **рҹҗһ Visual Debugger (DAP)** вҖ” A complete debugging interface for **Java, Go, Python, JavaScript/TypeScript, C, and C++** with a visual UI, breakpoints, step-over/into/out, and automatic JDK exit handling.
- **вҳ• Enterprise Java** вҖ” Deep `jdtls` integration with automated workspace management, build-on-attach, test discovery, and a smart DAP session that automatically terminates when your program exits.
- **рҹҡҖ Universal Code Runner** вҖ” Run any file instantly with `<leader>rr`. Supports Java, Python, C, C++, Go, Rust, JavaScript, TypeScript, Bash, and Lua вҖ” zero extra plugins required.
- **рҹҺЁ Dynamic Theming** вҖ” System-wide color synchronization via `pywal.nvim` with a built-in theme picker. Instantly switch between curated color schemes.
- **рҹ“Ҡ Live GitHub Dashboard** вҖ” The startup dashboard shows live GitHub notifications, open issues, PRs, and git diff status using the GitHub CLI.
- **рҹ—Ӯ Workspace & Session Management** вҖ” Jump between multiple projects instantly. Sessions are automatically saved and restored per-workspace.
- **рҹ–Ҙ Neovide Support** вҖ” First-class GUI support with zoom controls (`Ctrl++`, `Ctrl+-`, `Ctrl+0`).

### Required

These must be installed before running Neovim.

| Dependency | Purpose | Install |
| :--- | :--- | :--- |
| **Neovim** `v0.10+` | The editor itself | [neovim.io](https://neovim.io) |
| **Git** | Plugin management, Gitsigns | System package manager |

| **Java JDK 11+** | For Java/JDTLS support | [sdkman.io](https://sdkman.io) |

### Recommended (Optional)

| :--- | :--- | :--- |

| **Rust / cargo** | Rust code runner support | [rustup.rs](https://rustup.rs) |

### Auto-Installed by Mason

When you first open Neovim, `mason.nvim` will automatically install these:

- **LSP Servers:** `html-lsp`, `css-lsp`, `tailwindcss-language-server`, `typescript-language-server`, `lua-language-server`
- **Formatters:** `stylua`
- **Debug Adapters:** `java-debug-adapter`, `java-test`, `codelldb` (C/C++/Rust), `debugpy` (Python)

## рҹҡҖ Installation

> **Back up your existing config first!**
> ```bash
> mv ~/.config/nvim ~/.config/nvim.bak
> ```

**1. Clone the repository:**
```bash
git clone https://github.com/CralVivy/cedarvim ~/.config/nvim
```

**2. Launch Neovim:**
```bash
nvim
```
`lazy.nvim` will automatically bootstrap and install all plugins on first launch. This may take a minute.

**3. Authenticate GitHub CLI** *(for live dashboard):*
```bash
gh auth login
```

**4. Done.** Mason will install LSP servers automatically when you first open a supported file type.

## рҹ“Ӯ Architecture

CedarVim uses a strict separation of concerns. The `kickstart/` layer provides the stable base; the `custom/` layer is where all CedarVim-specific logic lives.

```
~/.config/nvim/
в”ңв”Җв”Җ init.lua                    # Core options, keymaps, lazy.nvim bootstrap, Telescope
в”ңв”Җв”Җ ftplugin/
в”Ӯ   в””в”Җв”Җ java.lua                # JDTLS auto-attach & DAP configuration for Java
в””в”Җв”Җ lua/
    в”ңв”Җв”Җ kickstart/plugins/      # Base layer (stable, rarely modified)
    в”Ӯ   в”ңв”Җв”Җ debug.lua           # nvim-dap + dapui + all language adapters
    в”Ӯ   в”ңв”Җв”Җ gitsigns.lua        # Git hunk signs + inline blame
    в”Ӯ   в”ңв”Җв”Җ neo-tree.lua        # File explorer
    в”Ӯ   в”ңв”Җв”Җ indent_line.lua     # Indentation guides
    в”Ӯ   в”ңв”Җв”Җ lint.lua            # Async linting
    в”Ӯ   в””в”Җв”Җ autopairs.lua       # Auto bracket/quote pairing
    в””в”Җв”Җ custom/                 # CedarVim's core logic layer
        в”ңв”Җв”Җ keymaps.lua         # All custom keybindings
        в”ңв”Җв”Җ runner.lua          # Universal code runner (10 languages)
        в”ңв”Җв”Җ theme_switcher.lua  # Dynamic theme picker
        в”ңв”Җв”Җ toggle_buffer.lua   # Buffer management utilities
        в”ңв”Җв”Җ utils.lua           # Shared helper functions
        в”ңв”Җв”Җ configs/            # Granular plugin-specific settings
        в”Ӯ   в””в”Җв”Җ bufferline.lua
        в”ңв”Җв”Җ plugins/            # Plugin specifications
        в”Ӯ   в”ңв”Җв”Җ lsp.lua         # Mason + LSP server setup
        в”Ӯ   в”ңв”Җв”Җ blink.lua       # blink.cmp + Copilot + LuaSnip
        в”Ӯ   в”ңв”Җв”Җ codecompanion.lua # AI chat + inline actions
        в”Ӯ   в”ңв”Җв”Җ workspaces.lua  # Project/workspace management
        в”Ӯ   в”ңв”Җв”Җ themes.lua      # Color scheme collection
        в”Ӯ   в”ңв”Җв”Җ buffer-ui.lua   # Bufferline configuration
        в”Ӯ   в””в”Җв”Җ init.lua        # Misc plugins (toggleterm, pywal, etc.)
        в””в”Җв”Җ snacks/             # Unified snacks.nvim configuration
            в”ңв”Җв”Җ dashboard.lua   # Startup dashboard + GitHub integration
            в”ңв”Җв”Җ picker.lua      # File/grep/LSP picker setup
            в”ңв”Җв”Җ indent.lua      # Indent scope visualization
            в””в”Җв”Җ snacks.lua      # Core snacks module toggles
```
